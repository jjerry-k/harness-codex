# Architecture Patterns

## 1. Pipeline
순차 의존 작업에 사용합니다.

적합한 경우:
- 앞 단계 결과가 다음 단계 입력일 때
- 구현 흐름이 직선적일 때

## 2. Fan-out / Fan-in
독립 작업을 병렬로 수행한 뒤 마지막에 통합합니다.

적합한 경우:
- 분석/구현 단위를 병렬화할 수 있을 때
- 프론트와 백엔드처럼 분리 가능한 흐름일 때

## 3. Expert Pool
상황에 따라 필요한 전문가만 선택 호출합니다.

적합한 경우:
- 모든 역할이 항상 필요하지 않을 때
- 라우팅이 중요한 문제일 때

## 4. Producer-Reviewer
초안을 만든 뒤 별도 검토자가 검증합니다.

적합한 경우:
- 산출물 품질이 중요할 때
- 생성과 검수를 분리해야 할 때

## 5. Supervisor
중앙 오케스트레이터가 작업 분배와 상태를 관리합니다.

적합한 경우:
- 작업이 복잡하고 중간 조정이 중요할 때
- 여러 하위 흐름을 지속적으로 관리해야 할 때

## 6. Hierarchical Delegation
상위 계획자가 하위 작업 단위로 계속 위임합니다.

적합한 경우:
- 작업 규모가 크고 계층 분해가 필요할 때

## 빠른 선택 가이드
- 단순 순차 작업: Pipeline
- 병렬 분석 후 합치기: Fan-out / Fan-in
- 선택적 호출: Expert Pool
- 초안 + 검수: Producer-Reviewer
- 복잡한 상태 관리: Supervisor
- 대규모 분해: Hierarchical Delegation
