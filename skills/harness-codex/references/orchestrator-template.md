# Orchestrator Template

오케스트레이터는 아래 순서로 결과를 정리합니다.

## 1. Harness recommendation
- 어떤 패턴을 쓸지
- 왜 이 패턴이 맞는지
- 대안 패턴이 왜 덜 적합한지

## 2. Role set
각 역할마다 정리합니다.
- role name
- mission
- responsibilities
- expected outputs

## 3. Execution flow
- phase-by-phase workflow
- 병렬 단계와 순차 단계 구분

## 4. Validation loop
- 무엇을 검증해야 하는지
- 무엇을 나중으로 미뤄도 되는지

## 5. Risks
- 가장 큰 리스크
- 리스크 완화 방법

## 6. Ready-to-use prompts
- orchestrator용 프롬프트
- 각 역할용 프롬프트
